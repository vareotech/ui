# AGENTS

## Projeto

`vareo-design-system` e a base visual e de componentes compartilhados da vareo.tech.

## Diretrizes

- Trate `src/tokens`, `src/foundations`, `src/components` e `src/templates` como a espinha dorsal do sistema.
- Antes de expandir variantes, documente a regra visual no Storybook.
- Prefira APIs simples, estados explícitos e semântica consistente.
- Toda decisão transversal de tema, acessibilidade ou composição deve ser refletida nos tokens antes de espalhar classes locais.
- Quando trabalhar em componentes ou stories deste projeto, consulte o MCP `vareo_storybook` para validar documentação, stories e contratos visuais atualizados.
