# Input

Input is the base text field for search, registration, and filtros, always paired with a visible label for acessibilidade.

## When to use

- Capture single-line text such as nomes, emails, or termos de busca.
- Combine with helper text and validation messaging for clarity.
- Reinforce context by grouping fields with related labels.

## Variants

| Variant | Purpose |
| --- | --- |
| Default | Standard editable state for user input. |
| Readonly | Surface data without allowing edits. |
| Disabled | Indicate an action is not available. |
| Invalid | Highlight errors via invalid state and messaging. |

## Do's and Don'ts

### Do's

- Always use visible labels to describe each field.
- Group related fields by context, alinhando verticalmente se necessário.
- Provide helper text and validation messages próximos ao input.

### Don'ts

- Don't hide the label or rely only on placeholder text.
- Do not overload a single input with too many responsibilities.

## Usage

```jsx
<Input label="Nome" helperText="Use o nome completo" />
```
