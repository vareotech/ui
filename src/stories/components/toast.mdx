# Toast

Toast confirms outcomes briefly without blocking the interface.

## When to use

- Use for lightweight success or error feedback after an action.

## Variants

| Variant | Purpose |
| --- | --- |
| Success | Confirms completion. |
| Error | Signals a failed outcome. |

## Do's and Don'ts

### Do's

- Keep the message concise.

### Don'ts

- Don't put long instructions inside a toast.

## Usage

```jsx
toast.success('Saved successfully')
```
