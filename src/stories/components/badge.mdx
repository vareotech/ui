# Badge

Badge surfaces short status and metadata in dense interfaces.

## When to use

- Use to mark status, category, or compact metadata.

## Variants

| Variant | Purpose |
| --- | --- |
| Primary | Main highlighted status. |
| Secondary | Lower emphasis metadata. |
| Success | Positive state. |
| Warning | Cautionary state. |
| Destructive | Critical state. |

## Do's and Don'ts

### Do's

- Keep badge labels short.

### Don'ts

- Don't use badge for long sentences.

## Usage

```jsx
<Badge variant="success">Synced</Badge>
```
