# Empty State

Empty State explains absence and suggests the next useful action.

## When to use

- Use when data is missing but the interface remains healthy.

## Variants

| Variant | Purpose |
| --- | --- |
| Default | Explains why the page is empty and what to do next. |

## Do's and Don'ts

### Do's

- Offer a clear next action.

### Don'ts

- Don't leave the area blank without context.

## Usage

```jsx
<EmptyState title="No orders yet" actionLabel="Create order" />
```
