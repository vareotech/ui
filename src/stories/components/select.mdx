# Select

Select orchestrates controlled choices with accessible focus, composed of trigger, valor, content, and items to keep interactions consistentes.

## When to use

- Let users choose a single option from a predefined lista.
- Surface focus states and ensure keyboard navigation stays previsível.
- Flag errors with aria-invalid when selection is required.

## Variants

| Variant | Purpose |
| --- | --- |
| SelectTrigger | Opens the dropdown region. |
| SelectValue | Displays the chosen option. |
| SelectContent | Wraps the list of SelectItem. |
| SelectItem | Represents each option with focus feedback. |

## Do's and Don'ts

### Do's

- Keep Select controlled and sync value with the form state.
- Preserve consistent focus order for accessibility.
- Apply aria-invalid to communicate errors.

### Don'ts

- Avoid mixing multiple select patterns on the same row.
- Don't hide the selected state; always show the current value.

## Usage

```jsx
<Select aria-invalid={hasError}>
  <SelectTrigger />
  <SelectContent>
    <SelectItem value="1">Option 1</SelectItem>
  </SelectContent>
</Select>
```
