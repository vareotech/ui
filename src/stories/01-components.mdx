# Component Catalog

Every component in the Vareo DS acts as a product contract — purpose, anatomy, states, interaction, correct usage and anti-patterns. Use the sidebar to navigate directly to each component, or browse the categories below.

Each component now has a **Docs tab** (prose documentation) and visual **Stories** for interactive exploration.

## Page structure

Each component page follows this structure:

1. **Docs tab** — When to use, variants, Do's/Don'ts, usage examples and props reference.
2. **Overview story** — Visual anatomy with live components.
3. **Interaction stories** — Play functions testing focus, click, typing, opening.
4. **A11y** — Accessibility violations checked automatically on every test run.
